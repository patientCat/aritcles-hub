# 从零构建理解Agent-04-基础外界交互能力

到第3章为止，我们已经有了：

1. ReAct 推理循环
2. 可用的记忆模块（短期/长期/语义召回）
3. 可执行测试

但是还缺一层关键能力：**和外界环境交互**。

如果 Agent 只能“思考 + 读自己记忆”，却不能看时间、读文件、访问网页，那它在真实任务里会很快失效。

所以第4章的核心目标是先补齐基础环境交互层。

---

## 本章目标

构建一个最小但可扩展的外界交互能力层，至少覆盖：

1. 时间查询（系统时钟）
2. HTTP 读取（只读抓取）
3. 本地文件读取（受限路径）

并把这些能力接入现有 ReAct Agent。

---

## 你将得到什么

完成后，Agent 将具备以下行为：

1. 用户问“现在几点”，Agent 可调用工具返回真实时间
2. 用户给一个 URL，Agent 可抓取文本并基于内容回答
3. 用户让它查看某个白名单目录下文件，Agent 可读取并总结

---

## 工程原则（本章强制）

1. 默认只读，不做写操作
2. 明确权限边界（路径白名单、请求超时、最大返回长度）
3. 每个外界能力都可单测
4. Agent 层不直接碰底层 API，只通过能力注册层调用
