# 从零构建理解Agent-05-Skill体系与ToolManager重构

第4章补齐了基础外界交互能力之后，我们要解决一个新问题：

能力越来越多时，Agent 如何稳定地“选能力、组能力、控流程”？

第5章的目标是引入 Skill 体系，并重构 ToolManager，让能力管理从“平铺工具列表”升级为“可组合的执行单元”。

---

## 本章目标

1. 统一能力抽象：Tool 与 Skill 共存
2. 重构 ToolManager：支持元数据、权限、可观测性
3. 引入 Skill 编排：一个 Skill 可调用多个 Tool
4. 增加回归测试：覆盖能力选择与执行路径

---

## 预期结果

完成后，Agent 将具备：

1. 面向任务选择 Skill，而非只靠单工具调用
2. Skill 内部自动编排多步调用
3. 能力调用日志可追踪、可回放

---

## 与第4章关系

第4章提供“能力原子件”，第5章负责“能力组织方式”。

先把外界交互打牢，再谈 Skill 编排，才能避免做出只会“空转思考”的 Agent。
